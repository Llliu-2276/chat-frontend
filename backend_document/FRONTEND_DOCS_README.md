## 📊 项目概览

### 已实现功能模块

- ✅ **用户认证**：注册、登录、登出、Token管理
- ✅ **用户资料**：查看、修改用户名、修改密码
- ✅ **在线状态**：心跳机制、在线人数统计
- ✅ **用户搜索**：用户名/账号搜索、分页查询
- ✅ **好友系统**：好友列表、未读消息、聊天记录、好友申请（双向确认）
- ✅ **WebSocket**：实时消息推送、在线状态通知、已读回执、好友申请通知

### 计划中功能模块

- 🔄 **群组系统**：创建群组、群组管理
- 🔄 **群组 WebSocket**：群聊消息广播

## 💡 开发建议

### 1. 先跑通示例
项目提供了完整的前端示例页面：
```
frontend-examples/
├── login.html          # 登录页面
├── chat.html           # 聊天页面
├── my.html             # 我的信息
├── search.html         # 搜索用户
└── get-user-info.html  # 查看用户信息
```

建议先在浏览器中打开这些页面，体验完整流程。

### 2. 使用测试工具
- **Postman**：手动测试接口
- **浏览器开发者工具**：查看网络请求
- **PowerShell脚本**：自动化测试

### 3. 参考代码示例
文档中提供了大量的代码示例，可以直接复制使用：
- JavaScript原生fetch
- Vue.js (axios)
- React (fetch封装)

### 4. 注意认证流程
```
登录 → 保存Token → 启动心跳 → 携带Token请求 → 登出清除Token
```

这是最核心的流程，务必理解透彻。

## 📞 获取帮助

### 遇到问题时
1. **查阅文档**：90%的问题文档中都有答案
2. **查看示例**：`frontend-examples/` 中有完整示例
3. **检查日志**：浏览器控制台 + 后端日志
4. **使用工具**：Postman验证接口是否正常

### 常见问题
- ❓ 接口401错误 → Token未正确携带或已过期
- ❓ 跨域错误 → 后端已配置CORS，检查请求地址
- ❓ 数据为空 → 检查数据库是否有数据
- ❓ 心跳失败 → 检查Token是否有效
